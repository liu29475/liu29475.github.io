Python操作码——BUILD_MAP_UNPACK_WITH_CALL

发表于 2020-07-12 更新于 2020-07-15 分类于 python

BUILD_MAP_UNPACK_WITH_CALL 编号是151，是个带参数的操作码。

这个操作码在 Python3.5 中引入，作用与 BUILD_TUPLE_UNPACK_WITH_CALL 非常类似。后者将函数调用中传入的多个序列解包的参数合并为一个元组，而 BUILD_MAP_UNPACK_WITH_CALL 则是将多个映射解包合并为一个字典，对应诸如 func(**kwarg1, **kwarg2)这样的语法。一些代码示例如下：

PyDict_New 函数用于创建空字典，关于这个函数的分析参考 BUILD_MAP_UNPACK，创建失败会返回 NULL，所以如果它的返回值为 NULL 将跳转到错误处理。

往下，操作码的参数 oparg 是映射对象的数量，全部都已经被压入了值栈中。通过 PEEK 宏获取它们，接着依次将它们作为参数调用 _PyDict_MergeEx 函数，将它们全部合并到空字典中。_PyDict_MergeEx 调用失败时会返回-1，所以当其返回值小于0时，则其调用过程中可能引发了异常。此时，将进行异常匹配，如果是 AttributeError 异常，则调用 format_kwargs_mapping_error 函数设置 TypeError，这个函数的实现如下：

此时 oparg 为2，stack_pointer[-4] 刚好指向了 func。

如果是 KeyError 异常，则调用 PyErr_Fetch 获取异常的类型，异常值与回溯。然后，再根据不同的异常值设置不同提示的 TypeError 或者调用 PyErr_Restore 重新存储异常值。设置完异常值后，将跳转到错误处理。

最后，如果合并所有的映射对象都没有引发异常，则将这些映射对象从值栈中弹出，并将合并好的字典压入值栈供后续操作码使用。
